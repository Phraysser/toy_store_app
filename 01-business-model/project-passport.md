## 2. БИЗНЕС-КОНТЕКСТ

**Проблема:**
Для покупки игрушек родителям необходимо посещать множество физических магазинов или использовать разные онлайн-платформы, что требует значительного времени на поиск нужных товаров, сравнение цен и оформление заказов. Отсутствует удобный мобильный инструмент с единым каталогом, корзиной и быстрым оформлением заказов.

**Решение:**
Разработать мобильное приложение Toy Store, позволяющее пользователям просматривать каталог игрушек, осуществлять поиск по категориям и названиям, добавлять товары в корзину и оформлять заказы в едином интерфейсе с возможностью оффлайн-просмотра каталога.

## 3. ЦЕЛИ ПРОЕКТА

1. **Обеспечить удобный доступ к каталогу:** реализовать интуитивный мобильный интерфейс для просмотра, поиска и фильтрации игрушек по категориям.
2. **Упростить процесс покупки:** разработать функционал корзины и оформления заказа с минимальным количеством шагов.
3. **Обеспечить надёжное хранение данных:** создать серверную часть с REST API и базой данных для синхронизации каталога, заказов и пользовательских данных.

## 4. КЛЮЧЕВЫЕ ПОКАЗАТЕЛИ УСПЕХА

| Показатель | Целевое значение |
|------------|------------------|
| Время поиска и добавления товара в корзину | не более 2 минут |
| Количество экранов мобильного приложения | не менее 7 |
| Стабильность API: время ответа (95-й перцентиль) | менее 500 мс |
| Покрытие кода серверной части тестами | > 40% |
| Поддержка оффлайн-режима | просмотр каталога без сети |

## 5. КЛЮЧЕВЫЕ РИСКИ

| Риск | Стратегия реагирования |
|------|------------------------|
| Сложность реализации оффлайн-режима | Использовать Room Database для локального кэширования каталога с последующей синхронизацией при появлении интернета. |
| Проблемы с загрузкой изображений | Реализовать библиотеку Coil с кэшированием и fallback на placeholder при ошибках загрузки. |
| Неверная оценка трудозатрат на UI | Использовать стандартные компоненты Material Design 3 для ускорения разработки интерфейса. |
| Проблемы с аутентификацией | Использовать JWT токены с EncryptedSharedPreferences для безопасного хранения. |

## 6. СТЕК ТЕХНОЛОГИЙ

| Уровень | Технология |
|---------|------------|
| **Мобильное приложение** | Android (Kotlin), Jetpack Compose, Material Design 3, Hilt, Retrofit2, Room |
| **Серверная часть** | Java 17, Spring Boot 3.x, Spring Security, Spring Data JPA |
| **База данных (сервер)** | PostgreSQL |
| **Аутентификация** | JWT (JSON Web Tokens), BCrypt |
| **Загрузка изображений** | Coil (client), Multipart REST API (server) |
| **Документация API** | OpenAPI 3 (Swagger UI) |
| **Архитектура** | PCMEF (Presentation-Control-Mediator-Entity-Foundation) |
