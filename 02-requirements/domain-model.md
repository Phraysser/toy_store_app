## Описание сущностей

### Пользователь (User)
**Атрибуты:** id, username, password (хранится в хешированном виде BCrypt), role, createdAt.

**Правила:**
- Username должен быть уникальным в системе.
- Пароль должен содержать не менее 6 символов.
- Роль по умолчанию — `USER`.

### Игрушка (Toy)
**Атрибуты:** id, name, description, price, category, imageUrl, stock, createdAt, updatedAt.

**Правила:**
- Название игрушки обязательно.
- Цена не может быть отрицательной.
- Остаток на складе (stock) не может быть отрицательным.
- Категория — строковое поле для группировки товаров.

### Корзина (Cart)
**Атрибуты:** id, userId, toyId, quantity, createdAt.

**Правила:**
- Пользователь может иметь несколько записей в корзине (по одной на каждый товар).
- Количество товара не может быть отрицательным или нулевым.
- При удалении товара из каталога связанные записи в корзине удаляются каскадно.

### Заказ (Order) *(перспектива развития)*
**Атрибуты:** id, userId, totalAmount, status, paymentMethod, shippingAddress, createdAt.

**Правила:**
- Заказ может быть отменён только в статусе `CREATED`.
- Общая сумма рассчитывается как сумма (цена × количество) всех элементов.
- При оформлении заказа товары резервируются на складе.

## Связи между сущностями

| Связь | Мощность | Описание |
|-------|----------|----------|
| Пользователь → Корзина | 1 : N | Один пользователь имеет много записей в корзине |
| Игрушка → Корзина | 1 : N | Одна игрушка может быть в корзине у многих пользователей |
| Пользователь → Заказ | 1 : N | Один пользователь может оформить много заказов |
| Заказ → Элемент заказа | 1 : N | Один заказ содержит много элементов |
| Игрушка → Элемент заказа | 1 : N | Одна игрушка может быть во многих заказах |
