# Спецификации прецедентов

Детальное описание ключевых сценариев использования системы.

---

## UC2: Войти в систему

| Поле | Описание |
|------|----------|
| **ID** | UC2 |
| **Название** | Войти в систему |
| **Акторы** | Пользователь |
| **Краткое описание** | Пользователь вводит username и пароль, получает JWT-токен для доступа к API. |
| **Предусловия** | Пользователь зарегистрирован в системе. |
| **Постусловия** | Пользователь аутентифицирован, JWT-токен сохранён на устройстве. |

### Основной поток событий
1. Пользователь открывает приложение, отображается экран входа.
2. Пользователь вводит username и пароль.
3. Пользователь нажимает кнопку «Войти».
4. Мобильное приложение отправляет POST-запрос `/api/auth/login` с учётными данными.
5. Сервер находит пользователя по username в БД, сравнивает BCrypt-хеши паролей.
6. Сервер генерирует JWT-токен и возвращает его в ответе.
7. Приложение сохраняет токен в EncryptedSharedPreferences.
8. Приложение переходит на главный экран (каталог игрушек).

### Альтернативные потоки

**5а. Неверные учётные данные:**
- Сервер возвращает HTTP 401.
- Приложение отображает сообщение «Неверный username или пароль».
- Возврат к шагу 2.

**4а. Нет соединения с сетью:**
- Приложение отображает сообщение «Нет подключения к серверу. Проверьте интернет-соединение.»
- Прецедент завершается неуспехом.

## UC3: Просмотреть каталог игрушек

| Поле | Описание |
|------|----------|
| **ID** | UC3 |
| **Название** | Просмотреть каталог игрушек |
| **Акторы** | Пользователь |
| **Краткое описание** | Пользователь просматривает доступные игрушки с возможностью фильтрации и поиска. |
| **Предусловия** | Пользователь аутентифицирован в системе (или гостевой доступ разрешён). |
| **Постусловия** | Отображён список игрушек с актуальными ценами и наличием. |

### Основной поток событий
1. Пользователь открывает приложение или переходит на вкладку «Каталог».
2. Система отправляет GET-запрос `/api/toys` для получения списка игрушек.
3. Сервер возвращает пагинированный список товаров с информацией о наличии.
4. Система отображает игрушки в виде сетки (2 колонки) с изображениями, названиями и ценами.
5. Пользователь может применить pull-to-refresh для обновления данных.

### Альтернативные потоки

**3а. Оффлайн-режим:**
- Если нет соединения с сетью, система загружает данные из локального кэша (Room Database).
- Отображается индикатор «Режим оффлайн».
- Прецедент завершается успехом.

**3б. Пустой каталог:**
- Если товаров нет, система отображает сообщение «Каталог пуст. Загляните позже!»
- Прецедент завершается успехом.

## UC5: Добавить товар в корзину

| Поле | Описание |
|------|----------|
| **ID** | UC5 |
| **Название** | Добавить товар в корзину |
| **Акторы** | Пользователь |
| **Краткое описание** | Пользователь добавляет выбранную игрушку в корзину с указанием количества. |
| **Предусловия** | Пользователь аутентифицирован. Выбран конкретный товар. |
| **Постусловия** | Товар добавлен в корзину, количество обновлено. |

### Основной поток событий
1. Пользователь открывает карточку товара или нажимает кнопку «В корзину» из каталога.
2. Система отображает диалог выбора количества (по умолчанию — 1).
3. Пользователь выбирает количество и нажимает «Добавить».
4. Мобильное приложение отправляет POST-запрос `/api/cart/add?toyId={id}&quantity={qty}`.
5. Сервер проверяет наличие товара на складе (stock >= quantity).
6. Сервер создаёт или обновляет запись в корзине.
7. Система отображает уведомление «Товар добавлен в корзину».
8. Иконка корзины обновляется с отображением общего количества товаров.

### Альтернативные потоки

**5а. Недостаточно товара на складе:**
- Сервер возвращает ошибку с кодом 409 (Conflict).
- Приложение отображает сообщение «Недостаточно товара на складе. Доступно: {stock}».
- Возврат к шагу 2.

**4а. Ошибка сети:**
- Товар сохраняется локально в Room Database с флагом «ожидает синхронизации».
- Приложение отображает сообщение «Товар добавлен в корзину (синхронизация при подключении)».
- Прецедент завершается успехом.

## UC7: Оформить заказ

| Поле | Описание |
|------|----------|
| **ID** | UC7 |
| **Название** | Оформить заказ |
| **Акторы** | Пользователь |
| **Краткое описание** | Пользователь подтверждает заказ с товарами из корзины. |
| **Предусловия** | Пользователь аутентифицирован. Корзина не пуста. |
| **Постусловия** | Заказ создан, корзина очищена. |

### Основной поток событий
1. Пользователь переходит на экран корзины.
2. Система отображает список товаров, общую сумму и кнопку «Оформить заказ».
3. Пользователь нажимает «Оформить заказ».
4. Система отображает диалог подтверждения с реквизитами для оплаты.
5. Пользователь нажимает «Подтвердить».
6. Мобильное приложение отправляет POST-запрос `/api/orders`.
7. Сервер создаёт заказ, резервирует товары на складе, очищает корзину.
8. Сервер возвращает подтверждение с номером заказа.
9. Система отображает экран «Заказ успешно оформлен!» с номером заказа.

### Альтернативные потоки

**6а. Товар закончился во время оформления:**
- Сервер возвращает ошибку с кодом 409.
- Приложение отображает сообщение «Товар «{name}» больше недоступен. Корзина обновлена.»
- Возврат к шагу 1.

**6б. Ошибка сети:**
- Заказ сохраняется локально с флагом «ожидает отправки».
- Приложение отображает сообщение «Заказ сохранён и будет отправлен при подключении».
- Прецедент завершается успехом.
