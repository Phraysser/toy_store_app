# Оценка трудозатрат по модели COCOMO

## Модель COCOMO II (Basic)

COCOMO (Constructive Cost Model) — алгоритмическая модель оценки трудозатрат на разработку ПО по объёму кода.

## Расчёт по COCOMO Basic

Формула: `Effort = a × (KSLOC)^b`

Для органического типа проекта (небольшая команда, хорошо понятные требования):
- `a = 2.4`, `b = 1.05`
- KSLOC = 3.49 (тысяч строк)

```
Effort = 2.4 × (3.49)^1.05
Effort = 2.4 × 3.72
Effort ≈ 8.93 ≈ 9.0 человеко-месяцев
```

Продолжительность: `Duration = 2.5 × (Effort)^0.38`

```
Duration = 2.5 × (9.0)^0.38
Duration = 2.5 × 2.29
Duration ≈ 5.7 месяцев
```

## Сравнение с фактическими трудозатратами

| Показатель | COCOMO (расчёт) | Фактически |
|------------|-----------------|------------|
| Объём кода (SLOC) | 3490 | ~3490 |
| Трудоёмкость | 9.0 чел.-мес. | ~4.5 мес. (1 разработчик) |
| Продолжительность | 5.7 месяцев | ~4.5 месяцев |

**Примечание:** COCOMO рассчитывает для команды. Для одного разработчика, совмещающего учёбу, трудозатраты в пересчёте: ~180 часов, что соответствует методичке (~120 часов для Траектории В).
