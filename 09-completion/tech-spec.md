## 1. Общие сведения

### 1.1 Назначение системы

Мобильное приложение «Toy Store» предназначено для покупки игрушек через интернет-магазин. Приложение обеспечивает просмотр каталога товаров, поиск и фильтрацию по категориям, управление корзиной покупок и оформление заказов. Приложение синхронизирует данные с серверной частью через REST API и поддерживает работу в оффлайн-режиме.

### 1.2 Цель разработки

Разработать мобильное Android-приложение с серверной поддержкой, обеспечивающее:
- удобный каталог игрушек с поиском и фильтрацией;
- управление корзиной покупок;
- надёжное хранение данных с синхронизацией через REST API;
- работу без интернета с последующей синхронизацией.

## 2. Требования к функциональности

### 2.1 Аутентификация и авторизация

| ID | Требование |
|----|------------|
| FR-01 | Регистрация нового пользователя (username, пароль) |
| FR-02 | Аутентификация по username и паролю, выдача JWT-токена |
| FR-03 | Хранение JWT-токена на устройстве в EncryptedSharedPreferences |
| FR-04 | Выход из аккаунта с очисткой локальных данных |

### 2.2 Каталог игрушек

| ID | Требование |
|----|------------|
| FR-05 | Просмотр каталога игрушек в виде сетки (2 колонки) |
| FR-06 | Поиск игрушек по названию |
| FR-07 | Фильтрация игрушек по категории |
| FR-08 | Просмотр деталей товара (изображение, описание, цена, наличие) |
| FR-09 | Pull-to-refresh для обновления каталога |
| FR-10 | Синхронизация каталога с сервером при наличии соединения |

### 2.3 Корзина покупок

| ID | Требование |
|----|------------|
| FR-11 | Добавление товара в корзину с указанием количества |
| FR-12 | Просмотр списка товаров в корзине |
| FR-13 | Изменение количества товара в корзине |
| FR-14 | Удаление товара из корзины |
| FR-15 | Очистка всей корзины |
| FR-16 | Отображение итоговой суммы заказа |
| FR-17 | Оформление заказа (checkout) |

### 2.4 Профиль и настройки

| ID | Требование |
|----|------------|
| FR-18 | Просмотр профиля (username, роль, дата регистрации) |
| FR-19 | Переключение тёмной/светлой темы |
| FR-20 | Выход из аккаунта |

### 2.5 Синхронизация и оффлайн

| ID | Требование |
|----|------------|
| FR-21 | Работа приложения без интернета (просмотр каталога из кэша) |
| FR-22 | Автоматическая синхронизация при восстановлении соединения |
| FR-23 | При входе в аккаунт — полная загрузка данных с сервера |

## 3. Нефункциональные требования

| ID | Категория | Требование |
|----|-----------|------------|
| NFR-01 | Производительность | Время ответа API ≤ 500 мс (95-й перцентиль) |
| NFR-02 | Надёжность | Данные не теряются при отсутствии сети |
| NFR-03 | Безопасность | Пароли хранятся в виде BCrypt-хешей, токены — JWT HS256 |
| NFR-04 | Совместимость | Android 8.0 (API 26) и выше |
| NFR-05 | Объём | Приложение занимает не более 50 МБ |
| NFR-06 | Тестируемость | Покрытие кода серверной части тестами > 40% |

---

## 4. Технические ограничения

| Параметр | Ограничение |
|----------|-------------|
| Платформа клиента | Android (Kotlin, минимум API 26) |
| Платформа сервера | Java 17, Spring Boot 3.x |
| База данных (сервер) | PostgreSQL 14+ |
| База данных (клиент) | Room Database (SQLite) |
| Аутентификация | JWT (HS256, срок действия 1 ч) |
| Сеть | HTTPS для production; HTTP допустим при локальном запуске |
